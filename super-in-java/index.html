  1. 默认调用 super()：

    • 当你在子类的构造方法中没有显式调用 super() 时，Java编译器会自动在子类构造方法的第一行插入一个对父类默认构造方法 super() 的调用。
    • 这意味着父类必须有一个无参的（默认的）构造方法，否则编译器会报错。

  2. 父类没有默认构造方法：

    • 如果父类没有无参的构造方法，而是只有带参数的构造方法，那么子类的每个构造方法都必须显式调用父类的构造方法，并传递适当的参数。
    • 这是因为编译器无法自动插入 super()，因为父类没有无参构造方法可调用。

    示例：

  3. 总结：

    • 如果父类有无参构造方法，子类的构造方法可以选择性地调用 super()，或者让编译器自动插入。
    • 如果父类没有无参构造方法，子类的每个构造方法都必须显式调用父类的构造方法，并提供必要的参数。

这样做的原因是确保在创建子类对象时，父类的部分能够被正确初始化。如果不遵守这一规则，编译器将无法确定如何初始化父类，从而导致编译错误。
