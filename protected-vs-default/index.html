protected vs default

在Java中，**访问修饰符（Access Modifiers）**用于控制类、方法、变量等的可见性和访问权限。Java提供了四种访问修饰符：

  1. public：公共的，所有类都可以访问。
  2. protected：受保护的，同一包中的类和不同包中的子类可以访问。
  3. 默认访问级别（包私有，Package-Private）：当没有显式指定访问修饰符时，默认为包私有，仅同一包中的类可以访问。
  4. private：私有的，仅在定义它的类内部可见。

重点关注：protected 与 默认访问级别（省略访问修饰符）

下面详细解释 protected 和默认访问级别之间的区别：

1. 定义与语法

  • protected 修饰符

  • 默认访问级别（省略修饰符）

2. 可见性范围

访问修饰符 同一类中 同一包中 不同包的子类 不同包的非子类
public ✔️ ✔️ ✔️ ✔️
protected ✔️ ✔️ ✔️ ❌
默认（包私有） ✔️ ✔️ ❌ ❌
private ✔️ ❌ ❌ ❌

  • protected

    • 同一类中：可访问。
    • 同一包中：可访问。
    • 不同包的子类中：可访问（通过继承）。
    • 不同包的非子类中：不可访问。

  • 默认访问级别（省略修饰符）

    • 同一类中：可访问。
    • 同一包中：可访问。
    • 不同包的子类中：不可访问。
    • 不同包的非子类中：不可访问。

3. 示例解析

同一包中的类
  • 在 同一包 中，protected 和默认访问级别的成员都可以被访问。
不同包中的子类
  • 在 不同包 中，protected 成员可以通过继承（即子类）访问。
  • 默认访问级别的成员在不同包的子类中不可访问。
不同包中的非子类
  • 在 不同包 的 非子类 中，protected 和默认访问级别的成员都不可访问。

4. 设计意图与使用场景

  • protected：

    • 设计意图：允许子类继承和访问父类的某些成员，同时限制外部类的直接访问。
    • 使用场景：当你希望子类能够访问和修改父类的某些成员，但不希望其他非相关类访问这些成员时，使用 protected。

  • 默认访问级别（包私有）：

    • 设计意图：限制成员仅在同一包内可见，增强封装性。
    • 使用场景：当你希望在同一包内的类之间共享成员，但不希望包外的类访问时，使用默认访问级别。

总结

  • protected：

    • 允许在 同一包 中访问。
    • 允许 不同包的子类 访问。
    • 不允许 不同包的非子类 访问。

  • 默认访问级别（省略访问修饰符）：

    • 仅允许在 同一包 中访问。
    • 不允许在 不同包 中的任何类访问，无论是否为子类。

选择使用 protected 还是默认访问级别取决于你希望如何控制成员的可见性。protected 提供了更灵活的访问权限，尤其是在需要通过继承关系进行访问时，而默认访问级别则更严格，仅限于包内使用。
